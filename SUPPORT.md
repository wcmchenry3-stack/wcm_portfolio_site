# Support

## How to get help

**For bugs and feature requests:** open a GitHub issue using the appropriate
template in the relevant repository.

**For questions about a specific repo:** open a GitHub Discussion in that repo
(if Discussions are enabled) or a GitHub issue tagged `question`.

**For security issues:** do **not** open a public issue. Follow the process in
[`SECURITY.md`](SECURITY.md) to report privately.

## What to include

When opening an issue or discussion, please include:

- A clear description of what you're trying to do
- What you expected to happen
- What actually happened (error messages, logs, screenshots if applicable)
- Relevant environment details (OS, language version, browser, etc.)

## Response times

This is a small organization maintained by a limited team. We aim to:

- Acknowledge new issues within **48 hours**
- Triage and prioritize within **1 week**
- Resolve or respond with a plan within **30 days** for confirmed issues

Response times may vary — thank you for your patience.

## Out of scope

Support is limited to issues within the `wcmchenry3-stack` repositories.
General questions about third-party tools (GitHub Actions, Cloudflare, etc.)
should be directed to those projects' support channels.
